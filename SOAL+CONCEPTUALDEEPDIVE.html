CIA Triad — Fondasi Keamanan

Tiga pilar utama yang menjadi dasar seluruh ilmu keamanan informasi.

🔒
Confidentiality

Data hanya dapat diakses oleh pihak yang berwenang. Implementasi: enkripsi, access control.

✅
Integrity

Data tidak dimodifikasi tanpa otorisasi. Implementasi: hashing, digital signature.

🟢
Availability

Sistem dan data tersedia saat dibutuhkan. Implementasi: redundansi, backup, anti-DDoS.

KEY TERM

CIA Triad bukan singkatan dari Central Intelligence Agency — melainkan Confidentiality, Integrity, Availability. Ini adalah framework yang digunakan untuk mengevaluasi keamanan sistem.

Enkripsi: Simetris vs Asimetris

Enkripsi Simetris
  • • Satu kunci untuk enkripsi dan dekripsi
  • • Lebih cepat, cocok untuk data besar
  • • Contoh: AES, DES, 3DES
  • • Masalah: bagaimana berbagi kunci dengan aman?
Enkripsi Asimetris
  • • Dua kunci: public key (enkripsi) + private key (dekripsi)
  • • Lebih lambat, cocok untuk pertukaran kunci
  • • Contoh: RSA, ECC
  • • Digunakan di HTTPS, digital signature
COMMON PITFALL

Banyak yang terbalik: mengira simetris = dua kunci. Ingat: SIMETRIS = SATU kunci (seperti kunci gembok yang sama untuk buka dan kunci). ASIMETRIS = DUA kunci (public + private).

SQL Injection — Cara Kerja & Pencegahan

SQL Injection terjadi ketika input pengguna tidak divalidasi dan langsung dimasukkan ke dalam query SQL.

❌ Kode Rentan
// Kode berbahaya — input langsung digabungkan
$username = $_POST['username'];
$query = "SELECT * FROM users WHERE username = '" 
         . $username . "'";
// Jika input: admin' OR '1'='1
// Query jadi: SELECT * FROM users WHERE username = 'admin' OR '1'='1'
// Hasilnya: SEMUA data users dikembalikan! 🚨
✅ Kode Aman (Prepared Statement)
// Menggunakan Prepared Statement — aman!
$stmt = $pdo->prepare("SELECT * FROM users WHERE username = ?");
$stmt->execute([$_POST['username']]);
// Input selalu diperlakukan sebagai DATA, bukan bagian query
// SQL Injection tidak mungkin terjadi ✅

Cross-Site Scripting (XSS)

❌ Contoh Serangan XSS
// Penyerang mengirimkan komentar berisi skrip:
"Artikel bagus! <script>document.cookie='stolen='+document.cookie;</script>"

// Jika website tidak mem-sanitize output, skrip dieksekusi
// di browser setiap pengunjung yang membaca komentar
// Akibat: cookie/session pengguna lain bisa dicuri 🚨
PENCEGAHAN XSS

1. Escape output — gunakan fungsi htmlspecialchars() di PHP atau equivalentnya
2. Content Security Policy (CSP) — header HTTP yang membatasi sumber skrip
3. Input validation — tolak input yang mengandung karakter HTML berbahaya

Hashing vs Enkripsi vs Salting

// ENKRIPSI — bisa dikembalikan (reversible)
plaintext  → [encrypt(key)] → "x9#mK2p"
"x9#mK2p" → [decrypt(key)] → plaintext  // Bisa kembali!

// HASHING — TIDAK bisa dikembalikan (one-way)
"password123" → [SHA256] → "ef92b778..."
"ef92b778..." → ??? → TIDAK BISA kembali ke aslinya

// SALTING — tambahkan random string sebelum hash
"password123" + salt("xR9#m2") → hash → "a93f12..."
// Mencegah rainbow table attack!
// Dua user dengan password sama → hash BERBEDA
COMMON PITFALL

Password TIDAK BOLEH disimpan dengan enkripsi (bisa didekripsi jika kunci bocor). Password harus disimpan dengan hashing + salting menggunakan algoritma khusus seperti bcrypt atau Argon2.

Conceptual Deep Dive Q1

Prinsip Keamanan Saltzer & Schroeder (8 Prinsip)

Common Pitfall

Mengurangi endpoint API publik bukan "meningkatkan availability" — tujuannya adalah mengurangi exposure (attack surface). Semakin sedikit endpoint terbuka, semakin kecil permukaan serangan.

Konsep Ancaman: Threat vs Vulnerability vs Risk

Threat Agent = entitas yang MEMILIKI kemampuan DAN NIAT untuk mengeksploitasi kerentanan.

Ransomware vs CIA Triad

Ransomware mengenkripsi file → menyerang Availability (file tidak bisa diakses) dan Integrity (file dimodifikasi tanpa izin). BUKAN hanya Confidentiality.

Confidentiality

Data bocor. Ransomware biasanya TIDAK membocorkan (kecuali double extortion).

Integrity ✓

File dienkripsi = diubah tanpa izin. Melanggar Integrity.

Availability ✓

File tidak bisa dibuka. Melanggar Availability.

Algorithm & Logic Visualization Q1

Fail-Safe Defaults Logic

// ❌ MELANGGAR Fail-Safe Defaults
function checkAccess(user) {
  try {
    if (authService.verify(user)) return "GRANTED";
  } catch (err) {
    // Sistem crash → akses diberikan penuh! 🚨
    return "GRANTED";
  }
}

// ✅ MENGIKUTI Fail-Safe Defaults
function checkAccess(user) {
  try {
    if (authService.verify(user)) return "GRANTED";
    return "DENIED"; // default = tolak
  } catch (err) {
    return "DENIED"; // Error → TETAP TOLAK
  }
}

Defense in Depth — Segmentasi Jaringan

// Tanpa Defense in Depth (satu lapis)
Internet ──→ [Firewall] ──→ Public Server ──→ Database
                               ↑ jika server jebol, DB langsung kena

// Dengan Defense in Depth (berlapis)
Internet ──→ [FW 1] ──→ [DMZ] ──→ Public Server
                                     │
                             [Firewall 2 (Internal)]
                                     │
                                 Database

Conceptual Deep Dive Q2

Include vs Extend Relationship

<<include>>

Use case SELALU menyertakan use case lain. Wajib, bukan opsional.

<<extend>>

Use case bersifat TAMBAHAN dan KONDISIONAL. Hanya muncul pada kondisi tertentu.

Aggregation vs Composition

Aggregation (◇ diamond kosong)

Hubungan LEMAH. Bagian DAPAT eksis tanpa keseluruhan. Siklus hidup independen.

Composition (◆ diamond hitam)

Hubungan KUAT. Bagian MATI bersama keseluruhan. Bergantung penuh pada siklus hidup whole.

Conceptual Deep Dive Q3

OCL analysis: Precondition vs Postcondition

Crucial Case: Pelanggaran Keduanya

Jika precondition gagal tapi sistem tetap lanjut dan mengubah state (authenticated=true), maka PRECONDITION dan POSTCONDITION dilanggar. (Pre melarang jalan, Post tidak terpenuhi karena operasi tidak sah).

context Person::createLogin(pass : String)
pre:  pass.size() >= 12
post: self.isAuthenticated = true

// Jika pass cuma 6 char tapi sistem set true:
// → PRE dilanggar (syarat masuk tidak dipenuhi)
// → POST dilanggar (janji state tidak sah)
